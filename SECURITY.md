# Security Policy

## Supported Versions

This project provides security updates for the following versions:

| Version | Supported          |
| ------- | ----------------- |
| 1.x     | :white_check_mark: |
| 0.x     | :x:                |

> Update these version numbers as your project evolves. For a new project like drew.lol, just track the current release.

## Reporting a Vulnerability

If you discover a security vulnerability in drew.lol, please report it privately so it can be addressed safely.

**How to report:**
- Email: `security@drew.lol` (replace with your contact email)  
- OR submit via [GitHub Security Advisories](https://github.com/DrewWritesCode/drew.lol/security/advisories)

**What to include in your report:**
- Description of the vulnerability  
- Steps to reproduce  
- Affected version(s)  
- Any suggested fixes (optional)  

**Response policy:**
- You can expect a response within **48 hours**.  
- We will provide updates until the issue is resolved.  
- Do not publicly disclose the vulnerability until a fix is released.  

## Scope

This policy covers all code in the `drew.lol` repository, including:  
- Laravel backend (`app/`, `routes/`, `database/`)  
- Frontend assets (JS, CSS, Blade templates)  
- Configuration files (`.env.example`, `.php` configs)
